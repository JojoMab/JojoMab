# Bewerbungskontext: Thermal Process Monitor

Dieses Projekt zeigt, wie technische Messdaten softwareseitig verarbeitet, bewertet und dokumentiert werden können. Es ist bewusst als überschaubares Bewerberprojekt aufgebaut, damit Aufbau, Code und Nutzen schnell nachvollziehbar sind.

## Relevanz für technische Informatik

- Verarbeitung strukturierter Sensordaten
- Prüfung von Grenzwerten
- Aufbau einer einfachen Alarmlogik
- Automatisierte Tests mit pytest
- GitHub Actions als CI-Pipeline

## Relevanz für Maschinenbau

- Temperatur, Druck und Durchfluss als technische Prozessgrößen
- Berechnung einer vereinfachten Wärmeübertragungsrate
- Bewertung eines Anlagenzustands anhand von Kennzahlen
- Bezug zu Wärmetauschern, Drucksystemen und Prozessanlagen

## Relevanz für Mechatronik

- Verbindung von Messdaten, Softwarelogik und technischer Bewertung
- Einstieg in industrielle Überwachungssysteme
- Ergänzende C++-Komponente als systemnahe Grenzwertprüfung

## Gesprächseinstieg im Bewerbungsgespräch

Das Projekt eignet sich gut, um im Bewerbungsgespräch über folgende Themen zu sprechen:

- Warum Grenzwertprüfungen in Anlagen wichtig sind
- Wie Sensordaten validiert werden können
- Warum automatisierte Tests auch bei kleinen Projekten sinnvoll sind
- Wie Python und C++ in technischen Projekten zusammen eingesetzt werden können
- Wie aus Rohdaten ein verständlicher technischer Bericht entsteht
